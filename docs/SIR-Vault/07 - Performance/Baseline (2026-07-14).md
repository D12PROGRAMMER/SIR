## Reading guide

- Enumeration cost scales with desktop population and tree size; caps bound the worst case ([[Cache and Enumeration]]).
- Press latency is a **policy constant**: the 150 ms settle exists to observe post-action state ([[Flow - Press Action]]). Optimizing it means changing policy, not code efficiency.
- The interesting optimization surface at this baseline: binary size (unstripped, no LTO), build time, and dependency count. See [[Performance Model]] and the optimization comparison when present.
